# cn-portal-pages

一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面内容的存储与分发基础单元。

## 项目简介

本项目旨在集中管理和维护一批独立的 HTML 页面。这些页面通常用于展示信息、提供工具界面或作为轻量级 Web 应用的入口。仓库结构清晰，便于添加、更新和引用各个页面。

## 目录说明

```
cn-portal-pages/
├── pages/           # 存放所有独立 HTML 页面文件
├── assets/          # 公共资源文件（CSS、JS、图片等）
├── index.html       # 仓库首页（可选）
└── README.md        # 本文件
```

- `pages/`：核心目录，每个页面为一个独立的 HTML 文件或子文件夹。子文件夹内可包含该页面所需的资源。
- `assets/`：存放多个页面共享的样式、脚本或图片，减少重复引用。
- `index.html`：可选，用于展示页面列表或提供导航。

## 页面归档说明

所有归档的 HTML 页面均为独立编写，不依赖外部数据库或后端服务。页面内容可能包括：

- 信息展示页
- 交互工具页
- 静态内容集合

每个页面均可在浏览器中直接打开运行，无需额外环境配置。页面间的链接或跳转应保持相对路径，以便在本地或静态托管环境中正常工作。

## 维护说明

- **新增页面**：在 `pages/` 目录下创建新的 HTML 文件或子文件夹，并确保内部资源路径正确。
- **更新页面**：直接修改对应文件，注意保持与公共资源的兼容性。
- **删除页面**：移除对应文件或文件夹，并检查是否有其他页面引用。
- **资源管理**：避免在页面目录中存放与页面无关的文件，保持目录整洁。

本仓库由维护者不定期更新，欢迎通过 Issue 或 Pull Request 提出建议或修正。所有贡献需遵循仓库的目录结构和文件命名规范。
